# Memory Maker — VPE 教程：管理一场完整的会议周期

> **适用对象：** 教育副主席 (VPE) 以及任何负责会议内容编排的干部。
> **目标：** 逐步演示 VPE 在 Memory Maker 中需要操作的每一个页面和动作——从会前一周到会后一天。
> **最后更新：** 2026-06-22

Memory Maker 是俱乐部用于规划、运行和记录头马会议的 Web 应用。作为 VPE，你是会议**内容**的主要负责人——议程、角色分配、备稿演讲以及 Pathways 进度跟踪。SAA 负责**场地**（签到、唱票、抽奖）。本教程聚焦 VPE 的职责范围，并在两个角色有交集处给出提示。

完整周期分为四个阶段：

1. **阶段一 — 会前规划**（会前 7 天到会前 1 天）：创建会议、搭建议程、开放角色预订。
2. **阶段二 — 接待签到**（会议当天 18:30）：SAA 负责签到，VPE 只需确认会议已就绪、可以发布。
3. **阶段三 — 现场会议**（会议当天 19:00–21:00）：发布 → 启动 → 应对现场临时变更 → 结束。
4. **阶段四 — 收尾与记录**（会后当天到次日）：确认演讲记录、Pathways 进度、投票结果，归档会议。

---

## 阶段一 — 会前规划

这是 VPE 工作的主要阶段。目标是让会议在当晚处于"已发布、议程完整、角色已预订"的状态——这样现场你唯一要做的就是按下**启动**按钮。

### 1.1 打开议程页面

进入 **`/agenda`**（或在顶部导航点击 **议程 / Agenda**）。这里是所有会议相关操作的主页。页面顶部有会议选择器和状态组，显示当前会议的状态（未发布 / 未开始 / 进行中 / 已结束 / 已取消）。

如果想要的会议还不存在，先创建（见 1.2）。如果已经存在但仍是**未发布 (Unpublished)** 状态，可以继续编辑。

### 1.2 从模板创建会议

1. 在议程页面点击 **新建 / New**（或在右上角菜单中选择 **创建 / Create**）。
2. 填写表单：
   - **会议日期 (Meeting Date)** — 会议实际召开的日期。Memory Maker 会标记可疑日期（例如过于久远的过去日期）并要求确认。
   - **会议编号 (Meeting Number)** — 俱乐部内唯一的正整数。用于在改期后重新编号。
   - **会议标题 / 主题 (Meeting Title / Theme)** — 贯穿整场会议的主题（例如"故事之夜"）。
   - **副标题 (Subtitle)**（可选）— 显示在标题下方的副标题。
   - **会议类型 (Meeting Type)** — 例如：常规会议、特别会议、联合会议。
   - **媒体链接 (Media URL)** — 会议录制的链接（会前留空，会后补上）。
   - **模板文件 (Template File)** — 选择一个会议模板（例如"标准俱乐部会议"），决定默认的议程项目。
3. 点击 **保存 / Save**。Memory Maker 创建会议记录，并根据模板生成默认的 `SessionLog` 行（每个议程槽位一行）。

> **提示：** 如果俱乐部只有一个模板，可以忽略该字段——它会默认使用该模板。请求路径为 `/agenda/create`，需要 `MEETING_CREATE` 权限。

### 1.3 搭建议程

议程页面以表格形式展示会议环节，列包括**开始时间 (Start Time)**、**环节 (Session)**、**负责人 (Owner)**、**项目 (Project)** 以及状态标志。每一行都可以直接内联编辑。

常见的编辑操作：

- **新增或删除环节** — 使用表格底部的行控件。删除环节是安全的——只要还没有人预订该角色。
- **调整环节顺序** — 拖动行的拖拽手柄，或使用行内的上下箭头。
- **编辑环节的开始时间** — 点击时间单元格，输入新值（24 小时制的 `HH:MM`）。如果两个环节时间冲突，Memory Maker 会给出警告。
- **设置每日一词 (WOD)** — 页面顶部有一个 `wod-display` 区块，你在这里设置的值会显示在幻灯片上，并被会议中的 Memory Maker 使用。
- **设置 GE 模式 (GE Style)** — 编辑器顶部的 `ge_mode` 下拉框控制总评人的报告风格。选择俱乐部使用的风格即可。
- **编辑会议级字段**（标题、副标题、类型、每日一词、媒体链接、会议编号）— 在议程编辑器的头部区域修改。保存后通过 `POST /agenda/update` 提交变更。

> **提示：** 表格中红色徽标表示校验问题（时间冲突、必需角色未指定负责人、备稿演讲环节缺少项目）。在发布前请先解决这些红色标记。

### 1.4 为备稿演讲添加项目信息

备稿演讲环节会被标记为 `Valid_for_Project = true`。对于每个此类环节，必须绑定一个 **Pathways 项目**，这样该次演讲才能计入演讲者的级别进度。

1. 点击该行的项目单元格。
2. 选择演讲者的路径（例如 *Dynamic Leadership*）。
3. 选择具体项目（例如 *Level 1 — Ice Breaker*）。
4. 可选：填写一个标题。

如果演讲者还没有确定项目，请主动询问——VPE 不应猜测，因为错误的项目会污染他们的 Pathways 记录。如果演讲者暂时未决定，可以留空，在会议**启动 (Start)** 之前补上即可。

### 1.5 开放角色预订

议程看起来没问题后：

1. 在状态组中点击 **发布 / Publish**。Memory Maker 会弹出确认框：*"发布后将向所有会员开放角色预订和演讲。请确保会议主题和结构已最终确定后再发布。"*
2. 确认后，状态从 `unpublished` 变为 `not started`。

会议现在对所有会员在预订页面 (`/booking`) 可见。会员可以自行预订角色；当某个槽位已满时，系统会自动启用候补名单 (waitlist)。

> **注意：** 发布后，**会议编号**、**标题**、**副标题**、**类型**、**每日一词** 仍然可以编辑，但议程结构调整会更麻烦——如果需要在发布后增删环节，请在会议**启动 (Start)** 之前完成。

### 1.6 预订或分配角色

有两种角色分配路径：

- **会员自助预订（默认）：** 会员访问 `/booking`，选择会议，点击某个槽位。系统支持候补名单——已满的槽位显示"加入候补 / Join Waitlist"按钮而不是"预订 / Book"。
- **干部分配（你的操作）：** 在议程页面，点击某行的负责人单元格，选择一个联系人。在会前 24–48 小时用此方式填补空缺槽位，以及为特定演讲分配评估人。

如果会员在候补名单上而槽位被释放，当当前负责人取消时，候补名单上的下一位会被自动提升。如果你手动重新分配，原负责人会被移除且系统不会通知他们——请通过其他渠道告知对方。

> **提示：** 留意 `Planner` 页面 (`/planner`)。会员用 Planner 跟踪他们想在未来会议中承担的项目。作为 VPE，你不需要管理 Planner 条目，但会前 1–2 天浏览 Planner 可以了解会员打算承担哪些槽位——在匹配演讲者与项目时很有帮助。

### 1.7 生成幻灯片

在议程页面，点击 **幻灯片 / Slides**（或通过 `/agenda/ppt/<meeting_id>` 导出 PowerPoint）。Memory Maker 会生成一个包含议程、每日一词、主题和演讲者/项目信息的 `.pptx` 文件。请预览一遍——如果有时间，建议走一遍幻灯片切换流程，特别是计时员/选票统计环节前后。

### 1.8 会前一天的自查清单

过一遍以下清单：

- [ ] 议程中没有红色校验徽标。
- [ ] 每个备稿演讲都已绑定 Pathways 项目。
- [ ] 每个必需角色都有负责人（或有意留空——由 TME 在现场找人顶替）。
- [ ] 每日一词已设置，"每日一词" 演讲者知道词义。
- [ ] 幻灯片已生成并保存到笔记本上。
- [ ] 已与 SAA 确认当晚由谁负责发布/唱票（或者由你负责）。

如果以上都打勾，你就可以进入阶段三了。

## 阶段二 — 接待签到（18:30）

这一阶段由 SAA 负责。作为 VPE，你唯一要做的是**保持手机畅通**，并且**在 SAA 准备好之前不要发布/启动会议**。SAA 的完整操作清单见 `SAA_MemMaker_User_Manual.md`。

你在此阶段可能要做的一件事：如果 SAA 让你临时将某位会员加入花名册，或将即兴演讲者加入议程——在议程页面直接操作即可，这些编辑在会议进行中也是允许的。

---

## 阶段三 — 现场会议（19:00 – 21:00）

议程已投到投影上，SAA 在门口迎宾，你在笔记本前。议程页面状态组中的状态按钮，是会议生命周期唯一的控制入口。

### 3.1 发布（如果还没发布的话）

如果之前因为议程未定跳过了 1.5，现在补上：在状态组中点击 **发布 / Publish**。会议处于 `unpublished` 时按钮显示"发布 / Publish"，点击后变为 `not started`。

> **注意：** 一旦发布，会员可以实时预订或取消角色。如果你必须做结构性变更（增删环节），请在发布前完成。

### 3.2 启动会议

在会议正式开始前约 5 分钟（通常 18:55）：

1. 打开 `/agenda` 并选择该会议。
2. 点击 **启动 / Start**。Memory Maker 将会议的 `Start_Time` 设为当前时间，状态从 `not started` 切换为 `running`。
3. 投票现已启用。会员可在 `/voting` 提交选票；SAA 在同一页面打开管理员开关 (admin toggle) 即可查看实时票数。

### 3.3 应对现场临时变更

会议进行中，你可能收到以下请求：

- *"能把 John 加为即兴演讲者 #2 吗？"* — 点击对应环节的负责人单元格，选择 John。会议处于 `running` 时该操作可用。
- *"Sarah 来不了，把她的 Ice Breaker 换成 Mike。"* — 点击负责人单元格，清除 Sarah，选择 Mike。系统无需刷新页面即可更新。
- *"现场加一个即兴演讲环节。"* — 使用 **新增环节 / Add Session** 控件。新行会追加到末尾；如需提前可拖动调整顺序。
- *"忘了分配评估人。"* — 与负责人编辑流程相同。评估人就是评估类环节的负责人。

> **提示：** 所有这些编辑都通过 `POST /agenda/update` 发送，一次往返即可完成校验和持久化。如果页面顶部弹出红色提示，说明修改被拒绝——阅读提示信息，修正数据后重试。

### 3.4 结束会议

当总评人结束发言（通常 21:00）：

1. 在状态组中点击 **结束 / Stop**。Memory Maker 会：
   - 将状态置为 `finished`。
   - 唱票并将获奖者写入投票记录（`_tally_votes_and_set_winners`）。
   - 删除该会议的所有候补名单条目。
   - 转换 `Planner` 状态：`booked` → `completed`，`waitlist` → `obsolete`。`draft` 保持不变。
   - 自动完成本次会议所有项目的 session log，并为每个负责人同步联系人元数据。
2. 投票关闭。SAA 现在可以在 `/voting` 查看最终结果，并在 `/luckydraw` 运行抽奖。

> **注意：** 一旦点击 **结束 / Stop**，会议即被封存，无法重新打开。如果误点了，请联系 ClubAdmin——唯一恢复路径是克隆会议或从备份恢复。

## 阶段四 — 收尾与记录（会后当天到次日）

会议结束，SAA 在收拾，你在安静的角落里花几分钟确保记录准确无误。

### 4.1 确认投票结果

打开 `/voting`（或 NPS 视图 `/voting_nps`）。你应该看到：

- 最终票数与各奖项获奖者。
- NPS 分数（如果俱乐部跟踪该指标）。
- **SAA 的抽奖核对名单**（用于确认抽奖对象正确——见 SAA 手册）。

如果某位获奖者看起来有误，请**不要**直接编辑选票记录。而是通过 `/issues` 提交一个问题单，以保留审计轨迹。

### 4.2 更新媒体链接

回到议程页面，在会议头部编辑 **媒体链接 (Media URL)** 字段并保存。这会将会议录屏关联到会议记录——会员以后可以在他们的 Planner 或会议档案中找到。

### 4.3 复核演讲记录

进入 **联系人 → 会员**（或 `/contacts/members`），打开每位完成备稿演讲的演讲者。确认：

- 项目记录正确。
- 级别/Pathways 进度正确。

由于会议结束时已自动完成项目，这一步基本是复核。如果发现异常（例如会议中临时更换了演讲者，错误的项目被自动完成），请通过 **演讲记录 / Speech Logs** (`/speech_logs`) 进行修正。

### 4.4 Pathways 进度

打开 `/pathways`，查看每位演讲者的路径。会议结束时已自动同步联系人元数据，理论上应能看到级别进度自动递增。如果没有，常见原因是该环节的项目被设置成了通用占位 (`ProjectID.GENERIC`)——改为真实项目后重新保存，会重新触发 `sync_contact_metadata`。

### 4.5 成就 (Achievements)

如果俱乐部使用成就模块 (`/achievements`)，查看本次会议是否产生了新徽章（例如"首次 Ice Breaker"、"最佳即兴演讲"）。通过成就页面授予。这是可选的，但会员会很开心。

### 4.6 归档或删除会议

议程页面的状态组现在显示的是 **删除 / Delete** 而不是 **结束 / Stop**。两种选择：

- **保留会议记录。** 它会显示在档案中，投票结果与演讲记录都被保留。大多数俱乐部的默认做法。
- **删除会议。** 需要 `MEETING_CREATE` 权限。这是硬删除（`meeting.delete_full()`）——选票、花名册、候补名单、Planner 条目和 session log 全部一并删除。仅当会议是误建且无真实数据时才这样做。

> **注意：** "删除" 按钮也用于移除重复或日期错误的会议。该操作不可撤销。

### 4.7 提前规划下一场

如果本次会议设立了新先例（新的环节顺序、新的模板、新的奖项类别），请更新**模板**和**设置**，让下一场会议搭建起来更容易。配置面参见 `template-manager-plan.md` 和 **设置 → 会议环节 (Settings → Sessions)**。

## 速查：状态流转图

```
未发布 (unpublished)  ──[Publish]──▶  未开始 (not started)  ──[Start]──▶  进行中 (running)  ──[Stop]──▶  已结束 (finished)
     │                       │                              │                              │
     │                       ▼                              ▼                              ▼
     │                  角色预订开放中                 投票进行中                    唱票完成
     │                  会员可预订                    候补名单在结束               候补名单清空
     │                  或候补                       时自动清理                   Planner→已完成
     │                                                                                项目自动完成
     │
     └────────────[Delete]───────────────────────────────────────────────────────[Delete]──▶ (已删除)
```

状态流转是单向的。要重做一场会议，只能新建。

## 权限速查表

本教程假设标准 VPE 角色拥有以下权限：

| 操作                                | 所需权限                  |
|-------------------------------------|---------------------------|
| 创建会议                            | `MEETING_CREATE`          |
| 编辑议程（环节、每日一词、项目）    | `MEETING_MANAGE`          |
| 分配 / 移除角色负责人               | `MEETING_MANAGE`          |
| 发布 / 启动 / 结束会议              | `MEETING_MANAGE`          |
| 删除已结束的会议                    | `MEETING_CREATE`          |
| 查看所有会议（含未发布）            | `MEETING_VIEW_ALL`        |
| 生成幻灯片 / 导出                   | `MEETING_MANAGE` + "Data/Slides Export" 模块 |

如果按钮是灰色的，说明你缺少对应权限。请联系 ClubAdmin 或在 **设置 → 用户 (Settings → Users)** 中检查。
