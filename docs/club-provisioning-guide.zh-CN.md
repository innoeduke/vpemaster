# 俱乐部配置指南 — 交接文档

一份友好的分步指南，帮助新任俱乐部管理员在系统中完成俱乐部的设置。前提是俱乐部管理员通过 Web 界面操作（无需使用命令行或编辑文件）。

---

## 第 1 部分：从俱乐部管理员处收集的信息

在登录系统前，请先从新任俱乐部管理员处收集以下信息。大部分都是**必填**的；所有信息**最终都会用到**，所以先尽可能收集，并对其余项安排后续跟进。

### A. 俱乐部身份（必填）

| 问题 | 示例 | 备注 |
|---|---|---|
| 贵俱乐部的 **Toastmasters 俱乐部编号** 是多少？ | `868941` | 必须唯一，由 Toastmasters International 分配 |
| 俱乐部的 **完整名称** 是什么？ | `Shanghai Leadership Toastmasters Club` | 将显示在所有议程和报告中 |
| 是否需要 **简称或缩写** 用于内部使用？ | `SLTMC` | 可选项；便于快速引用 |
| 贵俱乐部所属的 **District、Division 和 Area** 是什么？ | `District 85, Division K, Area K1` | 用于俱乐部信息横幅 |

### B. 会议安排（必填）

| 问题 | 示例 |

### F. 功能偏好（可选 — 可在使用过程中决定）

询问俱乐部管理员他们希望启用以下哪些功能。也可略过此部分，让他们使用中自行决定。

- [ ] **Booking**（角色预约）— 会员可自行预约未来会议中的角色？
- [ ] **Voting**（现场投票）— 会议现场由听众投票选出最佳演讲者、最佳评估者、最佳角色承担者？
- [ ] **Roster**（签到表）— 跟踪每次会议的出席情况？
- [ ] **Calendar**（日历视图）— 以日历形式查看会议？
- [ ] **Chatbot / AI Assistant**（AI 助手）— 由 Anthropic 驱动的俱乐部 AI 聊天？（需要 API key）
- [ ] **Journal / Speech Logs**（演讲日志）— 记录会员的演讲项目？
- [ ] **Club Directory**（俱乐部目录）— 将本俱乐部展示在跨俱乐部目录中？
- [ ] **Planner**（规划助手）— 帮助会员规划其角色履历？
- [ ] **Lucky Draw**（随机抽取）— Table Topics 的随机角色抽取？
- [ ] **Upload Links**（上传链接）— 俱乐部的文件收集？
- [ ] **Data / Slides Export**（数据 / 幻灯片导出）— 将议程导出为 Excel / PowerPoint？
- [ ] **Self Check-In**（自助签到）— 会员通过二维码 / 链接自助签到？

### G. 会议模板（可选）

系统预置了两个默认的会议模板（**Keynote Speech（主题演讲）** 和 **Panel Discussion（专题讨论）**）。请询问俱乐部是否使用这两个模板，或上传自定义模板。系统支持的完整模板列表：

- Keynote Speech（主题演讲）
- Panel Discussion（专题讨论）
- Speech Marathon（演讲马拉松）
- Speech Contest（演讲比赛）
- Debate（辩论赛）
- Pecha Kucha（佩查库查）
- Gavel Passing（移交仪式）
- Club Election（俱乐部换届选举）

## 第 2 部分：分步配置

收集完上述信息后，按以下步骤引导新任俱乐部管理员在 Web 界面中完成设置。

### 步骤 1：登录并创建俱乐部

1. 打开应用，以 **SysAdmin** 身份登录（开发者会提供初始设置的凭据）。
2. 导航到 **Clubs → New Club**。
3. 填写必填字段：
   - **Club Number**（来自 A 部分）
   - **Club Name**（来自 A 部分）
4. 点击 **Create Club**。系统会创建一个俱乐部空记录，并复制默认文件（徽标、幻灯片版式、两个默认议程模板）。

> 💡 **提示**：新建俱乐部初始时所有功能模块均为**禁用**状态，且没有 session type 或 role。接下来的步骤中会进行添加。

### 步骤 2：填写俱乐部详细信息

1. 导航到 **About Club**（或 **Settings → About**）。
2. 填写以下信息：
   - 简称、District/Division/Area（来自 A 部分）
   - 会议日、开始时间、地址、电话、官网（来自 B 部分）
   - 成立日期（来自 C 部分）
3. **上传俱乐部徽标**：
   - 点击徽标区域，选择你的 PNG / JPG / WebP 文件。
   - 系统会自动将其调整为 500×500 并转换为 WebP 格式。
4. 点击 **Save**。

> 💡 **提示**：俱乐部信息横幅（Club No. | Area | Division | District | Club Name）会根据这些字段自动生成。

### 步骤 3：创建首位管理员用户

> ⚠️ **此步骤需要 SysAdmin 权限** — 请与开发者协调。

开发者将通过命令行创建首位管理员用户，并在账户就绪后通知你。之后请：

1. 使用所提供的凭据登录。
2. 立即修改密码（**Profile → Change Password**）。
3. 确认你拥有 ClubAdmin 权限 — 导航到 **Settings**（应能看到所有选项卡）。

### 步骤 4：设置 ExComm 团队

1. 导航到 **Settings → ExComm** 选项卡。
2. 点击 **Add New Term**：
   - 输入 **任期代码（term code）**（例如 `26H1`）、开始日期、结束日期，以及可选的团队名称。
3. 保存后，你会看到 8 个职务。对每个职务，点击下拉框选择对应的会员。
4. 每次分配后保存。

> 💡 **提示**：如果下拉框中没有某位会员，需要先通过 **Members → Add Member** 添加。

### 步骤 5：启用功能模块

1. 导航到 **Settings → Permissions** 选项卡。
2. 滚动到 **Functional Modules** 部分。
3. 切换为 **ON**，启用俱乐部需要的功能模块（来自 F 部分）。Core 始终开启。
4. 点击 **Save**。

> 💡 **提示**：可随时更改。建议从核心功能（Booking、Voting、Roster）开始，待俱乐部熟悉后再逐步添加。

### 步骤 6：审阅并调整权限（可选）

1. 仍在 **Settings → Permissions**，滚动到 **Permission Matrix** 部分。
2. 审阅各角色的默认权限（SysAdmin、ClubAdmin、Operator、Staff、Member、Guest）。
3. 根据俱乐部工作流进行调整。
4. 点击 **Save**。

> 💡 **提示**：如果想把当前矩阵保存为本俱乐部的永久默认值，可点击 **Save as Default**。这会保存一个快照，以便后续 "Reset to Default" 时还原。

### 步骤 7：验证一切正常运行

在向会员正式介绍系统前，请快速做一次冒烟测试：

1. **创建一次测试会议**：进入 **Agenda → New**，填写信息后创建。
2. **打开议程**：确认俱乐部信息横幅、会议标题和日期均正确。
3. **检查一个模块**（如已启用）：尝试预约角色、投票，或俱乐部启用的任意功能。
4. **退出登录并以普通会员身份重新登录**：确认他们可以看到已发布的议程并预约角色。

## 常见陷阱

- **徽标非正方形**：非正方形徽标会被加黑边或裁剪。如有可能请提供正方形版本。
- **俱乐部编号已存在**：每个俱乐部编号在系统中必须唯一。创建前请向 Toastmasters 核实。
- **ExComm 任期日期**：Toastmasters 任期为半年。H1 = 7 月 - 12 月，H2 = 1 月 - 6 月。任期代码格式为 `<YY>H<1|2>`（例如 `26H1`）。
- **模块开关按俱乐部全局生效**：为俱乐部启用 "Voting" 并不等同于为每次会议启用 — 每次会议需单独发布并启动。
- **权限矩阵按俱乐部独立**：此处的更改仅影响本俱乐部，不影响系统中的其他俱乐部。

## 支持

如有疑问，请联系开发者。常见问题：

- **"Permission denied" 错误**：用户可能没有合适的角色。请检查 Settings → Members。
- **"Session type not found"**：俱乐部可能尚未植入 session type。开发者可运行 `flask import-data` 批量加载。
- **徽标无法上传**：文件可能过大（> 2MB）或格式错误。请尝试较小的 PNG 文件。
